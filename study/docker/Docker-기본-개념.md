# Docker 기본 개념

## 🤷🏻‍♂️ Docker를 왜 배우는 걸까?

Docker를 사용하는 가장 큰 이유는 "이식성"입니다. 즉, Docker를 활용하면 특정 프로그램을 다른 곳으로 쉽게 옮겨서 설치 및 실행할 수 있게 됩니다. 

예를 들어, MySQL을 A와 B 컴퓨터에서 각자 설치를 한다고 가정해 봅시다. A는 MySQL 설치를 끝낸 후, B에서 설치 방법을 알려주지만, B 컴퓨터에서는 먹히지 않을 수 있습니다. 그 이유는 각자의 컴퓨터 환경이 다 다르기 때문에 같은 방법을 해도 안 될 수 있게 됩니다. 그렇기 때문에 만약 이러한 상황 속에서 Docker를 활용한다면, 명령어 몇 줄로 MySQL 설치를 끝낼 수 있습니다.

정리하자면, 아래와 같습니다.

- 매번 귀찮은 설치 과정을 일일이 거치지 않아도 됩니다.
- 항상 일관되게 프로그램을 설치할 수 있습니다. (버전, 환경 설정, 옵션, 운영 체제 등)
- 각 프로그램이 독립적인 환경에서 실행되기 때문에 프로그램 간에 서로 충돌이 일어나지 않습니다.

## 🤷🏻‍♂️ IP와 Port란 무엇인가요?

IP는 네트워크 상에서의 특정 컴퓨터를 가리키는 주소이고, Port는 한 컴퓨터 내에서 실행되고 있는 특정 프로그램의 주소입니다. 

예를 들어 `13.250.15.132:8080`이라는 주소가 있다면, `13.250.15.132`이 IP에 해당하고 `8080`이 Port에 해당합니다.

IP와 Port 둘 다 알아야 하는 이유는, 현재 제 컴퓨터만 봐도 VsCode, 카카오톡, ChatGPT, 등 여러 프로그램이 켜져있습니다. 이는 실제 서버도 동일합니다. 하나의 프로그램만 띄워지는 것이 아닌, 여러가지 프로그램이 띄워지게 됩니다. 그렇기 때문에 만약 특정 인스턴스의 SpringBoot 서버와 연결을 하고 싶다면, 어떤 포트에서 돌아가고 있는지 알아야 합니다.

## 🤷🏻‍♂️ 그럼 브라우저 창에 포트 번호를 입력하지 않는 이유는 무엇인가요?

위에서 말한대로 라면, 주소창에 주소를 입력할 때 포트 번호를 알고 있어야 하는데 `google.com`를 입력할 때에는 포트 번호 없이 입력이 가능합니다. 그 이유는 기본적으로 브라우저는 80번 포트로 통신을 보내게 설정되어 있습니다. 그렇기 때문에 별도의 포트 번호 입력이 없더라도 접속이 가능합니다. 만약에 구글의 8071이라는 포트에 접속하고 싶다면, `google.com:8071`을 입력하면 됩니다.

## 🤷🏻‍♂️ Docker란 ? / 컨테이너(Container)란? / 이미지(Image)란? 

Docker란 컨테이너를 사용하여 각각의 프로그램을 분리된 환경에서 실행 및 관리할 수 있는 툴입니다. 그리고 컨테이너는 하나의 컴퓨터 환경 내에서 독립적인 컴퓨터 환경을 구성해서, 각 환경에 프로그램을 별도로 설치할 수 있게 만든 개념입니다.

컨테이너의 독립성은 아래와 같습니다.

- 디스크 (저장 공간) : 각 컨테이너마다 서로 각자의 저장 공간을 가지고 있다. 일반적으로 A 컨테이너 내부에서 B 컨테이너 내부에 있는 파일에 접근할 수 없습니다.
- 네트워크 (IP, Port) : 각 컨테이너마다 고유의 네트워크를 가지고 있다. 컨테이너는 각자의 IP 주소를 가지고 있습니다.

마지막으로 이미지는 프로그램을 실행하는 데 필요한 설치 과정, 설정, 버전 정보 등을 포함하고 있는 개념입니다. 즉, 프로그램을 실행하는 데 필요한 모든 것을 포함하고 있습니다.
